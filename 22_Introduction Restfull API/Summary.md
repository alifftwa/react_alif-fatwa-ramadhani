Rangkuman Materi Introduction to RESTful API:

1. **Understand the Concept of API:**

   - API (Application Programming Interface) adalah sekumpulan aturan dan protokol yang memungkinkan satu perangkat lunak untuk berinteraksi dengan perangkat lunak lainnya.
   - Memungkinkan aplikasi untuk menggunakan fungsionalitas atau layanan dari aplikasi lain.

2. **What is API?**

   - API adalah antarmuka yang memungkinkan aplikasi untuk berkomunikasi satu sama lain.
   - Memisahkan implementasi internal dari cara pengguna berinteraksi dengan aplikasi.

3. **REST (Representational State Transfer):**

   - Arsitektur web yang sering digunakan dalam pengembangan API.
   - Berfokus pada sederhana, mudah dimengerti, dan skalabilitas.
   - Menggunakan operasi HTTP standar seperti GET, POST, PUT, DELETE untuk berinteraksi dengan sumber daya (resources).

4. **JSON (JavaScript Object Notation):**

   - Format pertukaran data ringan dan mudah dibaca oleh manusia.
   - Digunakan dalam RESTful API sebagai format data yang umum digunakan.
   - Terdiri dari pasangan kunci-nilai yang dapat membentuk objek.

5. **HTTP Response Code:**

   - Kode respons HTTP memberikan informasi tentang hasil dari permintaan pengguna ke server.
   - Contoh: Kode 200 untuk OK, 404 untuk Not Found, dan 500 untuk Internal Server Error.
   - Membantu pemahaman apakah permintaan berhasil atau gagal.

6. **Open API:**
   - Open API (atau juga dikenal sebagai Swagger) menyediakan spesifikasi yang terbuka dan dapat diakses untuk API.
   - Memungkinkan pengembang memahami dan berinteraksi dengan API tanpa harus mengetahui detail implementasinya.
   - Dokumentasi yang mudah dimengerti dan dapat diakses.

Materi ini memberikan landasan pemahaman tentang API, menggali konsep REST, menjelaskan penggunaan format data JSON, membahas respons HTTP untuk pemahaman hasil permintaan, dan mengenalkan Open API sebagai alat dokumentasi yang berguna. Ini adalah pengetahuan dasar yang penting untuk bekerja dengan RESTful API dalam pengembangan perangkat lunak modern.
